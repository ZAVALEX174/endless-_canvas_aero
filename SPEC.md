# SPEC — Редизайн aero4 согласно model.pdf

Дата: 2026-04-09
Источник: собеседование по model.pdf

---

## 1. Контекст

Приложение представляет собой веб-инструмент для расчёта вентиляционных сетей шахт. Текущий UI:
- Хедер 40px
- Левый тулбар 52px (иконки)
- Контекстная полоска 38px (текстовые кнопки над холстом)
- Холст (Fabric.js)
- Правая панель 240px (3 таба: Свойства / Результат / Объекты)
- Статус-бар 24px
- Модальные окна для редактирования свойств линий и объектов

## 3. Правая панель — новая структура

### 3.1 Новые табы

Текущие табы (Свойства / Результат / Объекты) заменяются на:

| Таб | Содержимое |
|---|---|
| **Вентиляторы** | Библиотека изображений: только вентиляторы |
| **Сооружения** | Библиотека изображений: здания, станции |
| **Объекты** | Библиотека изображений: всё остальное |
| **Слои** | Управление слоями (новая функция) |

Каждый таб библиотеки — визуальная сетка (`image-library-grid`) как сейчас в табе «Объекты», но отфильтрованная по категории.

  `samohodnoe.png`, `zapasvhod.png`, `prohod.png`

**Объекты** (всё остальное):

- Перемычки: `petemichkabrick.png`, `petemichkaconcrete.png`, `petemichkametal.png`, `petemichkawood.png`

### 3.3 Таб «Слои»

Полная система управления слоями. Содержимое таба:

```
[+ Добавить слой]

┌──────────────────────────────────┐
│ 👁 Слой 1        [Rename] [Del]  │
│ 👁 Слой 2        [Rename] [Del]  │
│    Слой 3        [Rename] [Del]  │  ← глаз серый = скрыт
└──────────────────────────────────┘
```

**Функциональность:**
- **Создать слой** — кнопка «+ Добавить слой», создаёт с именем «Слой N»
- **Переименовать** — inline-редактирование по клику на имя или кнопке
- **Удалить** — кнопка удаления, объекты на слое переходят на «Слой по умолчанию»
- **Вкл/выкл видимость** — иконка глаза, скрывает/показывает все объекты слоя на холсте (Fabric.js `.set('visible', bool)`)
- **Слой по умолчанию** — «Основной» — существует всегда, нельзя удалить

**Данные:** каждый объект/линия хранит `layerId` в своих `properties`. При сохранении/загрузке слои сериализуются вместе с проектом.

## 4. Модальное окно — редизайн

### 4.1 Триггер открытия

| Элемент | Старый триггер | Новый триггер |
|---|---|---|
| Линия (ветвь) | Кнопка «Свойства линии» в контекст-баре | **Двойной клик** по линии на холсте |
| Объект (изображение) | Кнопка «Свойства объекта» в контекст-баре | **Двойной клик** по объекту на холсте |

Старые кнопки «Свойства линии» и «Свойства объекта» из контекстной полоски **удалить**.

### 4.2 Размер окна

Текущее модальное окно: ~420px ширина.
Новый размер: **650–720px** ширина, высота — по содержимому (с прокруткой если не помещается).

### 4.3 Новые поля

Добавляются в форму свойств линии и объекта:

| Поле | Тип | Поведение |
|---|---|---|
| **Номер** | `<input type="number">` | Автоинкремент при создании, редактируемый пользователем |
| **Слой** | `<select>` | Выпадающий список из созданных слоёв. По умолчанию — «Основной» |
| **Статус/Тип выработки** | `<select>` | Список типов: например, штрек, уклон, ствол, квершлаг и т.д. |
| **Отображение** | `<select>` | Уже есть в логике, добавить в форму (если отсутствует) |

### 4.4 Секция «Модельные параметры» — редактируемые поля

Поля Расход (м³/с), Скорость (м/с), Сопротивление (Н·с²/м⁴):

**UX:**
1. По умолчанию — `disabled` (readonly-вид), показывают значения из последнего расчёта
2. Рядом с секцией — кнопка **«Редактировать»**
3. Клик по «Редактировать» → поля становятся `enabled`, кнопка меняется на «Зафиксировать» и «Отмена»
4. **«Зафиксировать»** — сохраняет введённые значения как ручные, после чего они больше не перезаписываются автоматическим расчётом; поля снова disabled; визуально помечаются иконкой «замка»
5. **«Отмена»** — сбрасывает ввод, возвращает к авторасчёту
6. Дополнительная кнопка **«Сбросить вручную»** (появляется если поля зафиксированы) — снимает фиксацию и возвращает к авторасчёту
7. В заголовке секции подпись: «Считает само, но можно исправить вручную»

**Реализация:** хранить на объекте `properties.manualFlow`, `properties.manualVelocity`, `properties.manualResistance`. Если `manual` значение задано — расчётный движок не перезаписывает его (аналогично `isFlowSource` у вентилятора).

## 5. Левый тулбар — кнопка вентилятора

### 5.1 Новая кнопка

Добавить в тулбар (в группу управления расчётом или отдельно) кнопку:

```html
<button class="toolbar-btn" id="fanAnimationBtn"
        data-tooltip="Анимация вентиляторов (Реверс)"
        onclick="toggleFanAnimationAndReverse()">
  <!-- SVG: лопасть вентилятора (крыльчатка) -->
</button>
```

**SVG-иконка:** крыльчатка (пропеллер, 3–4 лопасти). При нажатии:
1. На все объекты типа `fan` на холсте применяется CSS-класс `fan-spinning` → анимация вращения иконки
2. Одновременно вызывается `toggleFanReverse()` для перерасчёта

**CSS-анимация (`animations.css`):**
```css
.fan-spinning {
  animation: spin 1s linear infinite;
}
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}
```

Кнопка имеет состояние активности (класс `active`): нажата — вентиляторы крутятся в реверсе + анимация; отжата — обычный режим, анимация стоп.

## 7. Система слоёв — интеграция

### 7.1 Хранение данных

- Глобальный массив `window.layers = []` (или в отдельном модуле `layersManager.js`)
- Формат слоя: `{ id: String, name: String, visible: Boolean }`
- Объекты/линии хранят `properties.layerId`
- При сериализации проекта (`projectManager.js`) — слои включаются в JSON

### 7.2 Видимость на холсте

При переключении видимости слоя:
```js
canvas.getObjects().forEach(obj => {
  if (obj.properties?.layerId === layerId) {
    obj.set('visible', visible);
  }
});
canvas.requestRenderAll();
```

### 7.3 Инициализация

При первом запуске (или загрузке проекта без слоёв) — автоматически создаётся «Основной» слой. Все существующие объекты получают `layerId = 'default'`.

## 8. Поле «Номер» — автоинкремент

- Глобальный счётчик `window.elementCounter = 0` (или часть `main5.js`)
- При создании линии или добавлении объекта: `properties.number = ++window.elementCounter`
- В модалке поле `<input type="number">` — заполнено автоматически, редактируемо
- Значение отображается рядом с названием в строке узла (опционально)

## 9. Технические ограничения и риски

| Риск | Описание | Митигация |
|---|---|---|
| Ручные значения vs. солвер | Если Расход зафиксирован вручную, солвер должен это учитывать | Добавить флаг `properties.manualOverride = true`, солвер пропускает поле |
| Слои + undo/redo | Добавление/удаление слоя должно попадать в undo-стек | Расширить `saveToUndoStack()` на слои |
| Двойной клик vs. Fabric.js drag | Fabric.js по умолчанию перехватывает `dblclick` для редактирования текста | Проверить конфликты в `canvasSetup.js` |
| Анимация вентилятора | Fabric.js рендерит объекты через canvas, CSS `transform` на canvas-элемент не работает | Реализовать анимацию через `fabric.util.requestAnimFrame` или через отдельный overlay-DOM-элемент поверх объекта |
| Производительность слоёв | Скрытие большого числа объектов через `requestRenderAll` может быть медленным | Использовать `canvas.renderAll()` один раз после batch-обновления |
