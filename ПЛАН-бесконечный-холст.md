# План: бесконечный холст + панорамирование (Space + drag)

> **Документ:** план реализации «бесконечного» холста с возможностью перетаскивания вида по зажатому пробелу.
> **Проект:** aero-redesign — калькулятор шахтной вентиляционной сети.
> **Дата:** 2026-04-14
> **Статус:** предложение, не реализовано.

---

## 0. Краткое резюме

Да, это **полностью возможно** на текущей архитектуре. Fabric.js нативно поддерживает пан/зум через `canvas.viewportTransform` — сдвигается только «окно просмотра», а все объекты остаются в своих мировых координатах. Это значит: расчёт воздуха, сплиты, рисование, сохранение в `.vnet` — **ничего из этого не сломается**, потому что вся геометрия уже работает в мировых координатах.

«Бесконечный» здесь — условно: физически canvas не растёт, но по нему можно ездить в любую сторону без ограничений, с сохранением позиций объектов в дальних координатах (например, `−5000, 12000`).

**Объём MVP:** ~150–200 строк кода. **Риск поломок:** низкий.

---

## 1. Почему это безопасно и что даёт Fabric из коробки

- `canvas.viewportTransform = [a, b, c, d, tx, ty]` — матрица 3×3 (в сокращённой форме). `tx`, `ty` — сдвиг вида; `a`, `d` — масштаб.
- `canvas.getPointer(e.e)` **автоматически** переводит экранные координаты мыши в мировые с учётом `viewportTransform`. Весь существующий код, использующий `getPointer`, продолжит работать без изменений.
- Объекты сцены хранят свои координаты (`left`, `top`, `x1/y1/x2/y2`) в **мировых** координатах. При пане меняется только отображение, сами объекты не перестраиваются.
- Сериализация/десериализация Fabric (`toJSON` / `loadFromJSON`) сохраняет мировые координаты — формат `.vnet` не затрагивается.
- Солвер (`airSolver.js`), `networkBuilder.js`, `getLineAbsoluteEndpoints` — всё это уже работает в мировых координатах и виртуально не зависит от viewport.

### Этап 1 — Ядро панорамирования

**Файл:** `js-for-index4/canvasSetup.js`

- [ ] **1.1.** Добавить глобальное состояние режима:
  ```js
  let isSpacePressed = false;  // флаг, что зажат пробел
  let isPanning = false;       // идёт ли сейчас перетаскивание
  let panOrigin = null;        // { clientX, clientY, vpt: [...] } в начале пана
  ```
- [ ] **1.2.** Обработчики клавиатуры (в `keyboard.js` или отдельно):
  - `keydown` `' '` (Space):
    - игнорировать, если фокус в `<input>` / `<textarea>` (уже есть паттерн)
    - `isSpacePressed = true`
    - `canvas.upperCanvasEl.style.cursor = 'grab'`
    - `e.preventDefault()` — чтобы страница не скроллилась
  - `keyup` `' '`:
    - `isSpacePressed = false`
    - Если шёл `isPanning` — завершить его принудительно
    - Вернуть курсор в `default` (или тот, что соответствует активному режиму)
- [ ] **1.3.** Интеграция с mouse-хендлерами Fabric:
  - `canvas.on('mouse:down', e)`:
    - если `isSpacePressed` → запомнить `panOrigin = { x: e.e.clientX, y: e.e.clientY, vpt: [...canvas.viewportTransform] }`, выставить `isPanning = true`, `canvas.selection = false`, курсор `grabbing`
  - `canvas.on('mouse:move', e)`:
    - если `isPanning` → вычислить `dx = e.e.clientX - panOrigin.x`, `dy = e.e.clientY - panOrigin.y`
    - обновить `canvas.viewportTransform[4] = panOrigin.vpt[4] + dx`
    - обновить `canvas.viewportTransform[5] = panOrigin.vpt[5] + dy`
    - вызвать `canvas.requestRenderAll()`
  - `canvas.on('mouse:up', e)`:
    - `isPanning = false`
    - вернуть `canvas.selection = true`
    - курсор → `grab` (если Space ещё зажат) или `default`
- [ ] **1.4.** Гард в существующих хендлерах:
  - В начале `handleCanvasMouseDown`, `handleCanvasMouseMove`, `handleLineDrawingEnd` и других добавить `if (isSpacePressed) return;` — чтобы при зажатом Space рисование/сплиты/драг объектов не срабатывали

### Этап 2 — Бесконечная сетка

Текущая сетка — `fabric.Group` фиксированного размера (`drawGrid()` в `canvasSetup.js`). После пана в дальние координаты она «закончится» — нужно либо пересчитывать, либо заменить на CSS-подложку.

#### Вариант A — CSS-фон (проще, MVP)

- [ ] **2.A.1.** Удалить из `drawGrid()` создание `grid-group` как fabric-объекта
- [ ] **2.A.2.** На `#canvas-wrapper` повесить фон:
  ```css
  #canvas-wrapper {
    background-image:
      linear-gradient(to right, rgba(255,255,255,0.04) 1px, transparent 1px),
      linear-gradient(to bottom, rgba(255,255,255,0.04) 1px, transparent 1px);
    background-size: var(--grid-size) var(--grid-size);
  }
  ```
- [ ] **2.A.3.** Синхронизировать сдвиг фона с `viewportTransform` в pan-handler:
  ```js
  const wrapper = document.getElementById('canvas-wrapper');
  wrapper.style.backgroundPosition = `${vpt[4]}px ${vpt[5]}px`;
  ```
- [ ] **2.A.4.** Учесть toggle grid (кнопка «Сетка» / клавиша `G`): `wrapper.classList.toggle('grid-off')` + CSS `.grid-off { background-image: none; }`

**Плюсы:** быстро, бесконечно из коробки, минимум кода.
**Минусы:** при будущем zoom нужно пересчитывать `background-size` пропорционально `viewportTransform[0]`.

#### Вариант B — Динамическая fabric-сетка (надёжнее для zoom)

- [ ] **2.B.1.** Перерисовывать `drawGrid()` на каждый pan/zoom, вычисляя visible rectangle:
  ```js
  const vpt = canvas.viewportTransform;
  const left   = -vpt[4] / vpt[0];
  const top    = -vpt[5] / vpt[3];
  const right  = left + canvas.width  / vpt[0];
  const bottom = top  + canvas.height / vpt[3];
  ```
- [ ] **2.B.2.** Рисовать линии сетки только в этом прямоугольнике + запас (например, +1 шаг с каждой стороны)
- [ ] **2.B.3.** Пересчитывать на `after:render` или в debounce-обёртке pan-handler

**Плюсы:** работает с zoom корректно, ничего не надо менять позже.
**Минусы:** больше кода, небольшая нагрузка на каждый pan.

**Рекомендация:** начать с **варианта A**. Перейти на B, когда будет добавлен зум.
